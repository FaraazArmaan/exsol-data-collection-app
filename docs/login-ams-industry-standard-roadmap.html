Current System Snapshot

This is the baseline this roadmap starts from. The system is already stronger than a basic MVP, but it still misses several controls expected in mature SaaS auth/admin.

Already strong

  • Separate admin and workspace-user session cookies.
  • Argon2 password hashing with dummy verify for timing equalization.
  • Google sign-in is strict-binding only, no auto-provisioning.
  • DB-refetched permissions and L1 Owner bypass.
  • Cross-client and subtree authorization helpers.
  • Integration tests cover many auth and AMS edge cases.

Below industry bar

  • No MFA or WebAuthn.
  • No server-side session revocation.
  • No CSP/HSTS/frame security header block.
  • Temporary plaintext passwords are stored until revealed or changed.
  • Admin impersonation does not carry admin attribution into every downstream write.
  • Platform admins are not least-privilege RBAC actors yet.

Roadmap rule

Each milestone should land as one thematic commit. After each milestone: update this HTML checklist, run targeted tests, run typecheck, and emit the house handoff for the Main integration chat. The human integrates and pushes.

Code economy is mandatory: think before coding. If the clear implementation is 10-20 lines, do not write 30-50. Keep each change isolated to the owning surface so a manual edit cannot crash an unrelated module or site section.

Risk Register

Keep this register live. If a risk becomes real, update the mitigation before the milestone handoff.

High: session migration

Adding revocation changes every protected request path.

Mitigate with characterization tests before refactor and staged helpers that preserve status codes.

High: impersonation attribution

Existing audit rows may only identify the impersonated Owner session.

Mitigate by extending the audit helper centrally and testing one representative downstream module write.

High: MFA lockout

Bad MFA rollout can lock out the bootstrap/admin owner.

Mitigate with recovery codes, bootstrap bypass only for enrollment recovery, and explicit tests.

Medium: CSP breakage

CSP can break Google Sign-In, inline styles, blob URLs, images, and local dev.

Mitigate with report-only locally first, then narrow production policy after browser smoke.

Medium: stale HR branch

The old Login+AMS worktree is really HR depth and predates current main cleanup.

Mitigate by creating the new hardening worktree from main and leaving HR untouched unless explicitly resumed.
